Context¶

Context is an object that contains the global state. Most functions in MuPDF take a context argument to be able to reference the global state.

Class API¶

class Context¶
__init__(self, max_store)¶

Constructor.

Allocate context containing global state. The global state contains an exception stack, resource store, etc. Most functions in MuPDF take a context argument to be able to reference the global state.

Parameters:max_store – maximum size in bytes of the resource store, before it will start evicting cached resources such as fonts and images. FZ_STORE_UNLIMITED can be used if a hard limit is not desired. Use FZ_STORE_DEFAULT to get a reasonable size.
get_aa_level(self)¶

Get the number of bits of antialiasing we are using. Between 0 and 8.

Return type:int
new_display_list(self)¶

Create an empty display list.

A display list contains drawing commands (text, images, etc.). Use DisplayList.new_list_device for populating the list.

Return type:DisplayList
new_pixmap(self, cs, w, h)¶

Create a new pixmap, with it’s origin at (0,0)

Parameters:
  • cs (ColorSpace) – The colorspace to use for the pixmap. See fz_device_rgb, fz_device_gray.
  • w (int) – The width of the pixmap (in pixels)
  • h (int) – The height of the pixmap (in pixels)
Return type:

Pixmap
new_pixmap_with_bbox(self, cs, bbox)¶

Create a pixmap of a given size, location and pixel format.

The bounding box specifies the size of the created pixmap and where it will be located. The colorspace determines the number of components per pixel. Alpha is always present.

Parameters:
  • cs (ColorSpace) – The colorspace to use for the pixmap. See fz_device_rgb, fz_device_gray.
  • bbox (BBox) – Bounding box specifying location/size of created pixmap.
Return type:

Pixmap
new_pixmap_with_bbox_and_data(self, cs, bbox, data)¶

Create a pixmap of a given size, location and pixel format, using the supplied data block.

The bounding box specifies the size of the created pixmap and where it will be located. The colorspace determines the number of components per pixel. Alpha is always present.
Parameters:
  • cs (ColorSpace) – The colorspace to use for the pixmap. See fz_device_rgb, fz_device_gray.
  • bbox (BBox) – Bounding box specifying location/size of created pixmap.
  • data (string) – The data block to keep the samples in.
Return type:

Pixmap

  • w (int) – The width of the pixmap (in pixels).

Pixmap
new_text_page(self, rect)¶

Create an empty text page.

The text page is filled out by the text device to contain the blocks, lines and spans of text on the page.

Parameters:rect (Rect) – The rectangle covering the page.
Return type:TextPage
new_text_sheet(self)¶

Create an empty style sheet.

The style sheet is filled out by the text device, creating one style for each unique font, color, size combination that is used.

Return type:TextSheet
open_document(self, filename)¶

Open a PDF, XPS or CBZ document.

Open a document file and read its basic structure so pages and objects can be located. MuPDF will try to repair broken documents (without actually changing the file contents).

Parameters:filename (String) – a path to a file as it would be given to open(2).
Return type:Document
set_aa_level(self, bits)¶

Set the number of bits of antialiasing we should use.

Parameters:bits (int) – The number of bits of antialiasing to use (values are clamped to within the 0 to 8 range).
